台東區營業處　電纜失竊調查暨損失計算系統　開發與修訂紀錄
v1.0 初版 2026年初 原始 Excel 系統
原始版本以 Excel 試算表為主要填報工具，作業流程由人工計算損失金額，表單格式固定不易調整，且無法在手機上便利操作。
v2.0 HTML 化 2026年初 轉換為單一 HTML 網頁應用程式
由線路課委託 AI Claude 協助，將 Excel 工作流程重新設計為單一 HTML 檔案，支援電腦與手機雙平台。主要功能包含：
  • 四大分頁：自主檢核表、失竊調查填寫、導線價格資料庫、服務所管理
  • 自動計算損失金額、損失材料合計、交通費
  • 導線品名下拉選單、變電所/饋線/轄區警局管理
  • localStorage 本機資料儲存
  • 手機響應式設計（底部導覽列、iOS 縮放防止）
  • 列印輸出：現場調查、計算表、附件資料、登記表四大區塊
  • PDF 檔名自動產生（依填報資訊命名）
v2.1 2026年初 列印優化與 UI 細節調整
針對實際列印輸出進行多次修訂：
  • 解決列印時出現空白首頁的問題
  • 字型統一為 PMingLiU／新細明體
  • 自主檢核表列印視窗排除手機導覽元素
  • 標題、簽名欄、章節標頭、出差費率、時間格式等排版細化
  • 新增登入圖示（藍底白燈泡）
  • 迴路名稱左至右快選槽（最多三組）
  • Android GPS 與相機權限提示
v3.0 雲端同步 2026年4月 導線資料庫改為 Google 試算表共用同步
各服務所人員使用私人手機，導線品名資料原本儲存於各自手機本機，造成資料各自獨立無法統一。本版本改以 Google 試算表作為共用資料來源：
  • 新增 SHEETS_CSV_URL 設定常數，填入 Google 試算表 CSV 發布網址即可啟用
  • 系統啟動時自動抓取最新導線清單（含四層 CORS proxy fallback 機制）
  • 頂部顯示同步狀態徽章（✅ 雲端已同步 / ⚠️ 使用本機快取 / ❌ 同步失敗）
  • 新增「🔄 重新同步」按鈕
  • 斷線時自動 fallback 至本機快取，不影響填表
  • 同步網址已設定為台東區營業處 Google 試算表
v3.1 2026年4月 導線價格資料庫改為唯讀顯示
導線資料統一由管理者維護 Google 試算表，HTML 端不再需要編輯功能：
  • 移除導線價格資料庫的「新增材料」按鈕
  • 移除各列「刪除」按鈕
  • 移除「💾 儲存所有價格」按鈕（含手機浮動按鈕）
  • 資料表格改為純文字唯讀顯示
  • 保留「🔗 開啟 NDCIS」與「🔄 重新同步」按鈕
v3.2 2026年4月 新增使用說明頁與修訂歷史頁
為方便日後接手人員了解系統背景，新增兩個資訊分頁：
  • 📌 使用說明：聯絡窗口（線路課巡修主辦）、填表前確認導線價格步驟
  • 📜 修訂歷史：記錄系統各版本演進
  • 標註系統由 AI Claude（Anthropic）協助製作
v3.3 2026年4月 Firebase Firestore 填報歷史功能
引入 Google Firebase Firestore 雲端資料庫，實現填報紀錄跨裝置共用：
  • 新增「📁 填報歷史」分頁，所有服務所人員均可查看所有紀錄
  • 表單頁新增「💾 暫存」按鈕，隨時儲存至 Firestore
  • 按「🖨 列印」時自動將紀錄存入歷史（狀態：已列印）
  • 歷史紀錄支援「📋 載入表單」，可將舊紀錄重新帶回填表頁修改
  • 修改與刪除功能受 PIN 碼保護
  • Firebase 專案：taitung-cable-theft（台東區公務帳號管理）
  • Firestore 安全規則設定為永久開放（內部系統，風險可接受）
  • Firebase Storage 需付費，照片上傳功能暫不實作
v3.4 2026年4月 PIN 碼改為 SHA-256 加密儲存
原本 PIN 碼以明文存放於 HTML 中，任何人用記事本開啟即可看到。本版改為加密方式：
  • PIN 碼改以 SHA-256 雜湊值儲存，無法從 HTML 原始碼反推原始密碼
  • 使用瀏覽器內建 Web Crypto API 在本機計算雜湊，原始 PIN 不離開裝置
  • 任何裝置輸入正確密碼均可通過驗證，跨裝置使用不受影響
  • 移除 PIN 輸入框的長度限制（原本限 6 位，實際需輸入 9 位）
v3.6 本版 2026年5月 填報功能優化與 UI 修正
根據實際使用測試，針對多項操作體驗問題進行修正：
  • 修正「經營線路單位」下拉選單在同步期間顯示空白的問題：將 UI 初始化移至雲端同步之前執行，表單開啟即可立即使用，不再需要等待同步完成
  • 修正「填報歷史」頁面的修改與刪除按鈕無反應的問題：改用 addEventListener 取代 innerHTML 內的箭頭函數 onclick，解決瀏覽器解析問題
  • 修正「載入表單」呼叫 clearForm() 時重複跳出確認視窗的問題：新增 clearFormSilent() 靜默清空函數供程式內部呼叫
  • 儲存流程改版：原「列印自動存檔」改為三個獨立按鈕（💾 暫存草稿、✅ 完成送出、🖨 列印），操作更直覺
  • 新增頂部「📋 無失竊案件」常駐按鈕，可隨時手動送出無失竊回報，不限每月1~5號才能觸發
  • 暫存資料完整性改善：collectFormData() 改存每列損失材料的品名、dbIdx 與數量，loadToForm() 改為完整還原動態列與自訂下拉選項
  • 失竊地點、桿號範圍、通報人欄位加入 lang="zh-TW" 屬性，預設切換中文注音輸入法
  • 損失材料與電表數量欄位從 value="0" 改為 placeholder="0"，輸入時不需先刪除預設值
v3.6 本版 2026年5月 修正手機版表格欄位對齊問題
修正導線價格資料庫與材料填寫表格在手機版顯示時欄位與數值錯位的問題：
  • 修正桌機版表格欄位標題對齊問題：導線價格資料庫（ptable）與材料填寫表格（etable）的數值欄標題原本靠左對齊，但儲存格數值靠右對齊，造成視覺錯位；修正為數值欄標題一律改為 text-align:right，與下方數值一致
  • 同步修正手機版橫向捲動問題：移除表格 display:block，改由父容器 .card overflow-x:auto 實現橫向捲動，維持表格正常欄寬渲染

2填寫前注意事項
發生導線失竊案件，填寫失竊調查表前請先確認導線價格是否為最新價格，避免計算金額有誤。

確認步驟：
① 切換至「💰 導線價格資料庫」頁面
② 按右上角「🔗 開啟 NDCIS」查詢最新單價
③ 若價格有異動，請通知主辦更新試算表
④ 按「🔄 重新同步」取得最新資料後再填表
